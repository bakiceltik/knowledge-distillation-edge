\section{Problem Definition}

The project studies whether knowledge distillation can produce a compact
plant-disease classifier suitable for edge or mobile deployment. The target
task is image-based classification on the PlantVillage dataset, where strong
teacher models can achieve high accuracy but may be too large or slow for
resource-constrained devices. The central research question is whether a
MobileNetV3-Small student can approach the performance of larger ResNet
teachers while reducing parameter count, model size, and inference latency.

The current report should be interpreted as a technical checkpoint. The
implemented experiment uses the three-class PlantVillage Potato subset for
fast validation of the data pipeline, model factory, evaluation code, and
reporting workflow. These results are not claimed as final full-dataset
performance.

\section{Dataset and Preprocessing Status}

The current experiment uses the PlantVillage Potato subset with three disease
or health categories: Early Blight, Late Blight, and Healthy. The subset
contains 1,721 RGB leaf images.
Because the downloaded dataset does not provide official train, validation,
and test splits, the pipeline creates a deterministic random split with seed
42.

All images are resized to $224 \times 224$ pixels and normalized with ImageNet
channel statistics. Training uses random horizontal flipping and small random
rotations, while validation and test transforms use only resizing and
normalization. Class labels are remapped after filtering so that the loss
function receives contiguous labels from $0$ to $C-1$.

\section{Literature Review Progress}

The literature review currently covers three project-relevant areas. First,
Mohanty et al.~\cite{plantvillage2016} showed that convolutional neural
networks can perform highly accurate image-based plant disease detection on
PlantVillage, while also motivating later work on generalization beyond
controlled image conditions. Second, Hinton et al.~\cite{hinton2015distilling}
introduced knowledge distillation, where a compact student model learns from
both hard labels and softened teacher predictions. The planned distillation
loss is

\begin{equation}
  \mathcal{L} =
  \alpha T^{2}\mathrm{KL}\!\left(
    \sigma\!\left(\frac{z_s}{T}\right)
    \,\big\|\,
    \sigma\!\left(\frac{z_t}{T}\right)
  \right)
  + (1-\alpha)\mathrm{CE}(z_s, y),
  \label{eq:distill}
\end{equation}

where $z_s$ and $z_t$ are student and teacher logits, $T$ is temperature,
$\alpha$ is the distillation weight, and $y$ is the ground-truth label. Third,
MobileNet and MobileNetV3~\cite{howard2017mobilenets,howard2019searching}
provide efficient architecture choices for edge deployment because they reduce
computation through depthwise separable convolutions and mobile-oriented
building blocks.

\section{Initial Experimental Results}

The current ResNet-18 experiment was read from the saved
\texttt{resnet18} Potato-subset output directory. It uses a pretrained ResNet-18
with 11,178,051 trainable parameters and evaluates on the 258-image Potato
test split. The saved metrics report 98.06\% test accuracy, 95.90\% macro-F1,
98.00\% weighted-F1, and 1.72 ms single-image inference latency on the
recorded device. These results are strong for a three-class subset, but they
should be treated as an initial pipeline checkpoint rather than evidence of
final full-dataset behavior.

\section{Planned Improvements and Technical Direction}

The next technical phase is to move from checkpoint validation to model
comparison. First, MobileNetV3-Small will be trained as a supervised compact
baseline. Second, ResNet-50 will be trained as a stronger teacher model.
Third, MobileNetV3-Small will be trained with the distillation objective in
Equation~\ref{eq:distill}. Initial distillation settings will use
$T=4.0$ and $\alpha=0.5$, then be tuned using validation macro-F1 and latency.

The comparison will include accuracy, macro-F1, weighted-F1, parameter count,
model size, and batch-1 inference latency. If time permits, post-training
INT8 quantization will be evaluated as an additional edge optimization method.

\section{Ablation and Error Analysis Plan}

The ablation plan is designed to isolate the contribution of architecture,
distillation, and compression choices:
\begin{itemize}
  \item Compare ResNet-18, ResNet-50, MobileNetV3-Small, and distilled
        MobileNetV3-Small under the same split and preprocessing pipeline.
  \item Vary distillation temperature $T$ and soft-target weight $\alpha$ to
        measure sensitivity of the student model.
  \item Compare supervised MobileNetV3-Small against distilled MobileNetV3-
        Small to quantify the effect of teacher supervision.
  \item Measure the effect of optional INT8 quantization on accuracy and
        latency.
\end{itemize}

The error analysis will inspect misclassified test images, per-class precision
and recall, confusion matrices, and confidence scores. Special attention will
be given to the Potato Healthy class, because its
small support makes recall more sensitive to a few errors.

\section{Current Challenges and Next Steps}

The main current challenge is that the Potato subset is small and visually
controlled, so it may overestimate performance compared with the full
38-class PlantVillage task and real-world field images. The healthy Potato
class is also much smaller than the disease classes, which makes class-level
recall less stable. A second challenge is to ensure fair latency measurement
across hardware backends, since CPU, MPS, and CUDA timings are not directly
comparable unless evaluated under a fixed protocol.

The immediate next steps are to finish the MobileNetV3-Small supervised
baseline, train the ResNet-50 teacher, run the first MobileNetV3 distillation
experiment, and expand evaluation from the Potato subset toward the full
PlantVillage dataset. The final comparison will focus on the accuracy,
macro-F1, parameter, model-size, and latency trade-off between teacher,
baseline, and distilled student models.
